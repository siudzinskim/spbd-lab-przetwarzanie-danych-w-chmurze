Modelowanie

dbt Cloud / Core

Używamy External Tables, aby BigQuery mógł czytać dane bezpośrednio z GCS bez ich fizycznego kopiowania.

GCS jako źródło dla BQ

Definiujemy źródła w sources.yml:


						sources:
						  - name: gcs_raw
						    schema: bookstore_src
						    tables:
						      - name: ext_customers
						      - name: ext_transactions

Umożliwia to query'owanie plików CSV/JSON bezpośrednio w SQL.

Zarządzanie Kluczami

Tworzymy inkrementalny model user_keys.sql:

  • Generowanie unikalnych Keysetów dla każdego użytkownika
  • Przechowywanie kluczy w dedykowanej warstwie Keystore
  • Zapewnienie niezmienności klucza raz wygenerowanego

						KEYS.NEW_KEYSET('AEAD_AES_GCM_256')

Szyfrowanie w Stagingu

Zamiast jawnych danych, w hurtowni ląduje szyfrogram:


						AEAD.ENCRYPT(
						  SAFE.FROM_HEX(k.encryption_key), 
						  first_name, ''
						)

Efekt: Nawet administrator hurtowni nie widzi danych PII bez dostępu do kluczy.

Kluczowe Techniki dbt

  • Incremental Materialization: Do bezpiecznego dopisywania nowych kluczy.
  • REPLACE operator: Eleganckie nadpisywanie wielu kolumn w jednym SELECT.
  • Complex Joins: Łączenie danych surowych z warstwą kluczy w locie.
  • Project Profiles: Konfiguracja uwierzytelniania OAuth do GCP.
