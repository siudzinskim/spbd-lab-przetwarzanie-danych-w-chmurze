Lab 01

Inżynieria Danych: Fundamenty i Generowanie Danych

Warsztaty: Przetwarzanie Danych w Chmurze

Cel Laboratorium

  • Pozyskanie i przygotowanie bazowego zestawu danych
  • Generowanie danych syntetycznych (Klienci, Transakcje)
  • Opanowanie podstaw DuckDB CLI
  • Zrozumienie różnicy między tabelami a widokami zewnętrznymi

Stack Technologiczny

Język skryptowy

Python 3

Silnik Analityczny

DuckDB

Formaty

CSV / JSON

DuckDB to nowoczesny, embedded silnik OLAP, który "zjada" CSV i JSON na śniadanie.

Start: Dane Bazowe

Wykorzystujemy realny zestaw danych z Kaggle (Books Sales):

  • Tytuły, autorzy, oceny, wydawcy.
  • Skrypt get_books.py przygotowuje plik books.csv.

Uwaga: To nasz fundament dla reszty warsztatów.

Symulacja Ekosystemu

Uruchamiamy generator.py, aby stworzyć:

  • customers.csv: Profil klienta (wiek, kraj, data rejestracji).
  • transactions.json: Zagnieżdżone dane o zakupach (items, ceny, ilości).

Technika kluczowa, gdy nie mamy dostępu do danych produkcyjnych na etapie deweloperskim.

Instalacja i Start


							curl https://install.duckdb.org | sh
							duckdb bookstore.ddb

Eksploracja Bazy

  • .tables / .schema – struktura bazy
  • FROM books; – skrócona kwerenda (specyfika DuckDB)
  • .import file.csv table – szybkie ładowanie

Kluczowa Lekcja: Storage

TABLE

Dane skopiowane do pliku .ddb. Szybkie, ale statyczne.

VIEW (External)

Dane czytane prosto z pliku JSON/CSV. Zawsze aktualne, ale zależne od pliku.


						CREATE VIEW v AS SELECT * FROM read_json('t.json');

Kroki Laboratorium

  1. Sklonowanie repozytorium i setup środowiska.
  2. Uruchomienie skryptów przygotowujących dane (Python).
  3. Instalacja i konfiguracja DuckDB CLI.
  4. Import danych klientów i transakcji (SQL).
  5. Eksperymenty z bazami In-Memory vs Persistent.
  6. Testowanie "odporności" widoków zewnętrznych na zmiany w systemie plików.

Podsumowanie Lab 01

  • Data Ingestion: Różne sposoby wprowadzania danych do silnika analitycznego.
  • Formaty plikowe: Praktyczne różnice między CSV a JSON w analityce.
  • DuckDB Mastery: Wykorzystanie nowoczesnego silnika do szybkiej eksploracji danych.
  • Persystencja: Zarządzanie bazami plikowymi i sesjami In-Memory.
  • Syntetyka: Budowa środowiska testowego przy braku realnych danych.
