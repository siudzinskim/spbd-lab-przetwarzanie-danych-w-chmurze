Orkiestracja

Airflow (Variables/Jinja)

Wykorzystujemy gsutil oraz google-cloud-storage Python SDK do komunikacji z chmurą.

Symulacja rzeczywistości

Uruchamiamy daily_data_generator:

  • Generowanie danych dla konkretnej daty wykonania (ds)
  • Zastosowanie unikalnych offsetów dla spójności danych
  • Automatyczne tworzenie plików klientów i transakcji

						templated_customers_output_file = f"customers-{{ ds_nodash }}.csv"

Struktura w GCS

Zamiast płaskiej listy plików, tworzymy hierarchię:


						gs://bucket/raw-data/customers/
						  ├── date=2024-01-01/
						  │     └── customers-20240101.csv
						  ├── date=2024-01-02/
						  │     └── customers-20240102.csv

Zalety: Szybsze filtrowanie, łatwiejsze odkrywanie danych (Data Discovery), zgodność ze standardami Big Data.

Airflow Variables

Zarządzanie konfiguracją bez zmiany kodu:


							GCS_BUCKET_NAME = Variable.get("GCS_bucket_name")

Jinja Power

Budowanie opcjonalnych flag komendy dbt:


							{% if dag_run.conf.get('dbt_models') %}
							  --select {{ dag_run.conf.get('dbt_models') }}
							{% endif %}

DuckDB + GCS + Python

W Lab 04 realizujemy ambitne zadanie:

  1. Pobranie danych partycjonowanych z GCS
  2. DuckDB: Scalenie wielu plików CSV przy użyciu hive_partitioning=TRUE
  3. Python: Logika biznesowa generowania nowych transakcji
  4. SDK: Przesłanie wyników z powrotem do chmury

Kroki Laboratorium

1. Konfiguracja połączenia z Google Cloud (GCP).

2. Ustawienie Zmiennych (Variables) w UI Airflow.

3. Uruchomienie generatora danych i weryfikacja w GCS.

4. Praca z dag_run.conf – ręczne sterowanie dbt.

5. Analiza logów procesów hybrydowych.

Czego nauczysz się w Lab 04?

  • Data Ingestion: Przenoszenie danych lokalnych do Cloud Storage.
  • Cloud Data Patterns: Implementacja Hive Partitioning w praktyce.
  • DuckDB Advanced: Czytanie danych prosto z chmury i obsługa partycji.
  • Airflow Customization: Wykorzystanie Variables i dynamicznej konfiguracji runtime.
  • Hybrid Operators: Łączenie Bash i Python w ramach jednego przepływu.
  • Modern ELT: Budowa potoków, które są skalowalne i łatwe w parametryzacji.
