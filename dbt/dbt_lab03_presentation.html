Lab 03

Orkiestracja dbt za pomocą Apache Airflow

Warsztaty: Przetwarzanie Danych w Chmurze

Cel Laboratorium

  • Automatyzacja procesów ELT
  • Uruchomienie Apache Airflow w trybie Standalone
  • Integracja Airflow z dbt za pomocą BashOperator
  • Budowa parametryzowanego potoku danych

Stack Technologiczny

Orkiestracja

Apache Airflow

Transformacja

dbt (core)

Baza Danych

DuckDB

Wykorzystujemy SQLite jako bazę metadanych Airflow dla uproszczenia konfiguracji.

Przebieg: Setup Airflow

Inicjalizacja środowiska skryptem ./init.sh:


							export AIRFLOW_HOME=/config/workspace/airflow
							./init.sh
							airflow standalone

Skrypt dba o migrację bazy danych, tworzenie użytkownika admin i konfigurację ścieżek.

Dostęp do UI

  • Host: http://[PUBLIC_IP]:8080
  • Logowanie: admin + hasło wygenerowane przy starcie
  • Weryfikacja: DAG server_health_check_empty

Potęga Parametryzacji

Tworzymy dbt_run_dag.py, który obsługuje:

  • Zmienne środowiskowe (ścieżki do projektu i profili)
  • Konfigurację runtime przez zmienne
  • Szablony Jinja do dynamicznego budowania komend bash

Struktura Pipeline'u

Zdefiniowany przepływ zadań:

seed >> run >> test >> docs
  1. Seed: Załadowanie danych statycznych
  2. Run: Wykonanie transformacji SQL
  3. Test: Weryfikacja jakości danych
  4. Docs: Generowanie dokumentacji

Kluczowe Techniki

  • Idempotentność: Każde uruchomienie daje ten sam wynik (CREATE OR REPLACE).
  • Inversion of Control: Airflow decyduje KIEDY, dbt decyduje JAK.
  • Symlink Management: Mapowanie folderów workspace do środowiska Airflow.
  • Log Analysis: Debugowanie procesów dbt bezpośrednio w logach Airflow.

Typowe Wyzwania

  • Błędy importu DAG-ów (list-import-errors)
  • Problemy ze ścieżkami względnymi i absolutnymi
  • Konieczność odświeżania (Clear task) po zmianach w kodzie

Czego nauczysz się w Lab 03?

  • Setup Airflow: Podstawowa konfiguracja środowiska Standalone z bazą SQLite.
  • BashOperator: Automatyzacja narzędzi CLI (dbt).
  • Jinja Templating: Budowa dynamicznych ścieżek i parametrów w DAG-ach.
  • Zarządzanie Stanem: Obsługa parametrów dag_run.conf i zmiennych środowiskowych.
  • Struktura Projektu: Wykorzystanie symlinków do łączenia workspace'u z systemem Airflow.
  • Log & Task Management: Analiza błędów dbt i sterowanie powtórzeniami (Clear Task).
